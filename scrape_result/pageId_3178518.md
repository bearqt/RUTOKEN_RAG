Объекты хранения не являются классом объектов, поэтому не имеют определения CKO\_  и включают в себя объекты данных ( [CKO\_DATA](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178520)), сертификатов ([CKO\_CERTIFICATE](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178523)), ключей ( [CKO\_PUBLIC\_KEY](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178529#id-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D0%BE%D1%82%D0%BA%D1%80%D1%8B%D1%82%D1%8B%D1%85%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9), [CKO\_PRIVATE\_KEY](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178529#id-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D0%B7%D0%B0%D0%BA%D1%80%D1%8B%D1%82%D1%8B%D1%85%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9), [CKO\_SECRET\_KEY](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178529#id-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9-%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D1%8B%D1%81%D0%B5%D0%BA%D1%80%D0%B5%D1%82%D0%BD%D1%8B%D1%85%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%B9)) и параметров домена ( [CKO\_DOMAIN\_PARAMETERS](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178531)).

Следующая таблица определяет общие атрибуты объектов хранения, в дополнение к [общим атрибутам](https://dev.rutoken.ru/pages/viewpage.action?pageId=3178516), определенным для этого класса объектов.

**Общие атрибуты объектов хранения**

| **Атрибут** | **Тип данных** | **Значение** |
| --- | --- | --- |
| CKA\_TOKEN | CK\_BBOOL | CK\_TRUE, если объект является объектом токена (т.е. будет создан на токене).<br>CK\_FALSE, если объект является объектом сессии (т.е. будет создан в оперативной памяти).<br>По умолчанию принимает значение CK\_FALSE |
| CKA\_PRIVATE | CK\_BBOOL | CK\_TRUE, если это закрытый объект. Значение атрибутов такого объекта можно получить только после ввода PIN-кода.<br>CK\_FALSE, если это открытый объект. Значение атрибутов такого объекта можно получить без ввода PIN-кода.<br>Значение по умолчанию определяется токеном и может зависеть от значения других атрибутов объекта |
| CKA\_MODIFIABLE | CK\_BBOOL | CK\_TRUE, если значения атрибутов объекта после его создания могут быть изменены.<br>CK\_FALSE, если объект создан «только для чтения» и значения атрибутов объекта после его создания не могут быть изменены.<br>По умолчанию принимает значение CK\_TRUE |
| CKA\_LABEL | RFC2279 string | Метка объекта (по умолчанию пусто) |
| CKA\_COPYABLE | CK\_BBOOL | CK\_TRUE, если объект может быть скопирован с использованием функции C\_CopyObject().<br>По умолчанию принимает значение CK\_TRUE. Не может быть изменен на значение TRUE после FALSE |

Только атрибут **CKA\_** **LABEL** может быть изменен после создания объекта. Атрибуты **CKA\_** **TOKEN**, **CKA\_** **PRIVATE** и **CKA\_** **MODIFIABLE** могут быть изменены в процессе копирования объекта.

Атрибут **CKA\_** **TOKEN** определяет, является ли объект объектом токена или объектом сессии.

Когда атрибут **CKA\_** **PRIVATE** принимает значение CK\_TRUE, пользователь не имеет доступа к объекту до тех пор, пока не выполнит аутентификацию на токене.

Значение атрибута **CKA\_** **MODIFIABLE** определяет, является ли объект доступным «только для чтения» или нет. Неизменяемый объект может быть удален.

Атрибут **CKA\_** **LABEL** предназначен для помощи пользователю при просмотре.

Значение атрибута **CKA\_COPYABLE** определяет, доступен ли объект для копирования или нет. Этот атрибут может использоваться в конъюнкции с **CKA\_** **MODIFIABLE** для предотвращения изменения разрешений использования ключа и других объектов.
